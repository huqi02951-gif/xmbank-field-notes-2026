# 《两岸实习手记》设计规范

## 目标与边界

本项目是一个面向台湾高校大三实习生的移动优先静态网站。它提供真实业务观察、三个业务瞬间、一个可操作的 APEX 轻量工具和少量下班后入口。它不是培训平台、任务中心、旅游攻略或银行产品营销页。

新网站只写入本独立仓库。APEX 原项目只读，无运行时依赖、软链接、子模块或反向同步。原 PDF 与内部资料仅放在被 Git 忽略的 `source-private/`。

## 产品结构

1. **首页**：“2026 · 厦门银行 / 两岸实习手记”，以抽象海岸线、城市轮廓和“银行—企业—城市”路线建立第一记忆点。
2. **银行里值得看的事**：三个可键盘展开的纵向问题，年份只作边注。
3. **三个真实业务瞬间**：分别使用对话、材料工作台和四问版式，避免三张同质卡片。末尾嵌入不存储、无输入框的“留下一句话”纸条。
4. **APEX 精选工具**：以“零散信息 → APEX 处理 → 清晰结果”建立第二记忆点。工具将 APEX 中已有的企业开户材料数据和清单格式化方式缩减为公开演示子集。
5. **下班以后**：仅提供徒步、咖啡/甜点/奶茶、辣口/漂亮饭三个轻入口。首版不造店名、评分、价格、排行或具体徒步路线。

## 视觉系统

方向为“夏日港口编辑手记”：米白纸面、深蓝墨色、少量厦门蓝和砖红/日光黄。字体只使用任务书指定的系统中文字体栈。桌面使用 12 列、手机使用 4 列，正文最宽 680px，内容容器约 1200px。

组件不共用统一“卡片皮肤”：问题区用分隔线，对话区使用编辑排版，材料区像工作桌，四问区使用大数字与留白，随手记是窄纸条，生活区是杂志末页。阴影仅用于必要浮层，圆角为 8–12px。

## 交互与状态

- 单页内部锚点导航，移动菜单不遮挡语言切换。
- 问题列表、APEX 示例和生活入口使用原生按钮与 `aria-expanded`，键盘可用，焦点可见。
- 动效仅限路线绘制、区块入场、APEX 切换和生活展开，并完整支持 `prefers-reduced-motion`。
- 复制提供“当前版本”主操作与简体/繁体次操作，成功或失败通过 `role="status"` Toast 告知。

## 简繁架构

`src/content/content.zh-CN.ts` 是唯一正文内容源。默认语言为 `zh-CN`，繁体显示时动态加载 OpenCC 兼容实现，使用台湾字形/词组配置。`src/i18n/bankingDictionary.ts` 在转换前占位保护机构名、品牌名、产品名和大陆正式业务术语，转换后恢复/修正。

语言选择写入 `localStorage`，同步 `document.documentElement.lang`。切换不调整滚动位置。当前页面、APEX 输出、复制内容和提示文字使用同一转换层，不回写原稿或 APEX 数据。

## APEX 公开适配

迁移边界为一个“企业开户材料助手”演示：用户只选择开户场景与已有材料，纯前端适配函数输出“准备材料 / 待补材料 / 业务理解提示”。页面不收集企业名称、证件号、手机号或财务数据，不保存、不上传，刷新即清空。

来自 APEX 的业务内容必须能够追溯到源文件和 Git 提交。公开版删除客户经理内部提醒、费用数字、内部系统路径和可能过期的绝对表述；输出始终附“示例仅用于理解材料结构，实际办理前请以当地网点最新要求为准”。

## 安全与发布

公开 Git 表面仅包含脱敏文案、源码、测试、可公开来源映射和生成的网页资产。`content-audit.mjs` 在构建与部署前扫描 PII、密钥、本地绝对路径、内部网址和过期产品数字，命中即非零退出。GitHub Actions 在 `main` 上执行安装、审计、测试和构建，再用官方 Pages Actions 发布 `dist/`。
