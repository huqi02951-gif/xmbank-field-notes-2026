# 内容维护说明

## 修改原则

1. 方法与学习内容可以更新，但不写入客户信息、内部操作路径或未经批准的对客承诺。
2. 涉及材料、规则、费用、时效、产品和办理结论时，必须由业务或合规负责人确认，并在文案中保留适用边界。
3. 简体是唯一人工维护版本；不要另建繁体内容副本。
4. APEX 教学数据只能从已批准、脱敏、可追溯的来源人工同步。

## 发布步骤

修改后先运行 `npm run verify`。所有检查通过后提交并推送到 `main`，GitHub Pages 工作流会自动部署。若线上结果异常，可在 GitHub Actions 中重新运行上一条成功提交，或将问题提交反向提交后再次推送；不要直接修改生成的 `dist/`。
